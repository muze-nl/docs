# Concept: Solid

Solid is a project with the intention of providing decentralised standards and protocols for the Web, as an alternative to the increasingly centralised direction taken by commercial web services. It comes from the Web’s inventor, Tim Berners-Lee, and its name is a portmanteau formed from SOcial LInked Data.

Central to Solid is the idea of moving a user’s data from centralised repositories to local ones belonging to that user. These repositories are called “pods”, from “Personal Online Data Store”, and can be hosted by the user themself or on a provider of their choosing. The user can then decide for themselves what data is stored in their pod, and to whom or to what it is shared.

On top of the pods, Solid provides a comprehensive vocabulary of protocols for authentication and communication between pods and applications, for the different tasks involved. All transactions between applications and users in a Solid system take place via these protocols, and no user data is stored anywhere but in the user’s pod.

Solid is intended to work with user data rather than application data. There is no reason why a Solid application can not use a pod of its own for application-level data, however many such applications may use more traditional storage means for non-user data.
